# Security and Anonymization

## Purpose

This repository is designed as a public portfolio case study. It documents the project approach, architecture and outcomes without exposing sensitive operational details.

## Data Not Included

- AWS account IDs
- real VPC, subnet or security group identifiers
- production region layout
- customer names
- internal application names
- production repository URLs
- internal billing exports
- raw Cost and Usage Reports
- secrets or access tokens
- real IAM role names
- confidential architecture diagrams

## Data Included

- anonymized project description
- representative architecture diagrams
- summarized financial metrics
- operating model and governance structure
- delivery timeline
- risk management approach
- lessons learned
- anonymized dashboard examples

## Anonymization Approach

| Sensitive detail | Portfolio representation |
|---|---|
| real application name | generic platform / workload name |
| exact account structure | generalized AWS environment |
| production IDs | omitted |
| raw billing data | summarized monthly values |
| team names | generic roles |
| internal meeting names | generic governance cadence |
